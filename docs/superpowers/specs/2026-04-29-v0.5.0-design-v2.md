### 3.1 Timezone & locale como configurações de plataforma

- Adicionar 2 chaves em `AppSetting` (categoria `platform`):
  - `platform.timezone` — default `"America/Sao_Paulo"`. Validação: deve ser IANA (`Intl.DateTimeFormat().resolvedOptions().timeZone` aceita).

- **Cache:** in-memory (`Map`) com TTL 60s. Invalidate explícito quando super_admin atualiza (server action chama `invalidatePlatformSettings()` antes de retornar).

### 3.3 Filtros simplificados

`PeriodKey` reduz de 6 → 4 valores: `hoje | semana_atual | mes_atual | custom`.

**Definições precisas:**
- `hoje`: 00:00:00 → 23:59:59.999 do dia atual no `platform.timezone`.
- `semana_atual`: segunda-feira 00:00 → próxima segunda-feira 00:00 no tz da plataforma. (ISO week, week start = Monday — padrão BR).
- `mes_atual`: dia 1 00:00 → dia 1 do próximo mês 00:00 no tz da plataforma.
- `custom`: usuário escolhe `start` e `end` (datas locais, convertidas para UTC range). Range máximo: 90 dias.

**UI:**
- Pills horizontais. Mobile: `overflow-x-auto` com scroll snap. Sem dropdown. Pill selecionada: `bg-primary text-primary-foreground`. Pill custom mostra ícone Calendar à esquerda + texto "Personalizado" ou range formatado quando ativo.
- Custom abre `Popover` em desktop (2 calendários inline lado a lado) e `Sheet bottom` em mobile (calendário stack vertical).
- Reset: clicar na pill ativa ou em "Limpar" reseta para `hoje`.

### 3.5 Senha temporária simples

`src/lib/utils/generate-temp-password.ts`:

```ts
const ALPHA = "abcdefghjkmnpqrstuvwxyzABCDEFGHJKLMNPQRSTUVWXYZ"; // sem i, l, I, L, o, O
const DIGITS = "23456789"; // sem 0, 1
const POOL = ALPHA + DIGITS;

export function generateTempPassword(length = 8): string {
  // Garante ≥1 letra e ≥1 dígito.
  const chars: string[] = [];
  chars.push(ALPHA[crypto.randomInt(0, ALPHA.length)]);
  chars.push(DIGITS[crypto.randomInt(0, DIGITS.length)]);
  for (let i = 2; i < length; i++) {
    chars.push(POOL[crypto.randomInt(0, POOL.length)]);
  }
  // Embaralha (Fisher-Yates) para não fixar letra+dígito nas posições 0/1.
  for (let i = chars.length - 1; i > 0; i--) {
    const j = crypto.randomInt(0, i + 1);
    [chars[i], chars[j]] = [chars[j], chars[i]];
  }
  return chars.join("");
}
```

- Default 8 chars. Sem caracteres confundíveis.
- Usa `crypto.randomInt` (Node, criptograficamente seguro).
- Substitui o generator antigo em `src/lib/actions/users/regenerate-password.ts`.

### 3.9 Mensagens não respondidas (#56)

Tela `/relatorios/mensagens-nao-respondidas`:
- Query: `src/lib/chatwoot/queries/mensagens-nao-respondidas.ts`
- Critério: `c.status=0` AND última `messages.message_type=0` (incoming) AND não há `messages.message_type=1` (outgoing) depois.
- KPIs no topo: total | tempo médio de espera | mais antigo (maior `now() - last_incoming_at`)
- Tabela: Nome | WhatsApp | Estado | Departamento | Atendente | Aguardando há | Snippet | Ações
- Ordenação default: aguardando há DESC (mais antigos primeiro)
- Filtros: período, inbox, team, atendente.

**Otimização:**
- SQL usa `LATERAL JOIN` ou subquery com `DISTINCT ON (conversation_id)` em messages para pegar a última mensagem.
- Sugestão de índice: `CREATE INDEX IF NOT EXISTS idx_messages_conv_created_type ON messages(conversation_id, created_at DESC, message_type)`. Como não temos write access ao Chatwoot DB, **documentar** o índice como recomendação (não criar via migration).
- Se query >2s, adicionar cache 30s.
