## 1. Contexto

Após v0.6.1, o usuário levantou problemas de qualidade no relatório principal (Conversas), navegação (sidebar com active state feio + bug) e na configuração do Agente Nex (modelos defasados, sem indicadores, sem busca, validação rasa). Esta release entrega **polimento e qualidade de experiência**, sem novos relatórios. Inclui também 2.0 da página de configuração do Agente Nex.

### Princípios

1. **Aproveitar telas grandes** — relatórios usam até 1600 px.
2. **Filtros mais inteligentes, menos grosseiros** — mais poder, menos ruído visual.
3. **Tour realmente útil** — cobre todas as funcionalidades, com layout íntegro.
4. **Sincronização real** entre toggle Matrix IA e todas as visualizações.
5. **Sidebar limpa e correta** — sem "borda esquerda violet"; sem ativar pai junto com filho.
6. **Branding mais curto**: "Relatórios Inteligentes".
7. **Agente Nex 2.0**: modelos atualizados, indicador de custo, busca, modelo livre, teste profundo, atalhos.

### 3.10 [R10] Sidebar bug — pais ativam junto

#### Hoje

```ts
function isActive(href: string) {
  if (href === "/dashboard") return pathname === "/dashboard";
  return pathname.startsWith(href);
}
```

Problema: pathname `/configuracoes/consumo` faz `startsWith('/configuracoes')` retornar true → ambos ficam ativos. Mesmo problema com `/perfil/trocar-senha` se `/perfil` é folha (que é).

#### Decisão

Implementar **longest-prefix-match** + **exact match para folhas**:

```ts
function buildActivePathChecker(items: NavItem[]) {
  // Coleta todos os hrefs raiz (sem children) numa lista ordenada por
  // especificidade (mais longo primeiro).
  const allLeafHrefs = collectLeafHrefs(items)
    .sort((a, b) => b.length - a.length);

  return function isActive(href: string, hasChildren: boolean) {
    if (hasChildren) {
      // Pai: ativo se algum child é ativo.
      // Implementação simplificada: pathname.startsWith(href + '/').
      return pathname.startsWith(href + "/") || pathname === href;
    }
    // Folha: ativo apenas se for o longest-prefix-match para o pathname.
    if (href === "/dashboard") return pathname === "/dashboard";
    if (pathname === href) return true;
    if (!pathname.startsWith(href + "/")) return false;
    // Verifica se existe outro href mais específico que é prefixo do pathname.
    const moreSpecific = allLeafHrefs.find(
      (h) => h !== href && h.startsWith(href + "/") && pathname.startsWith(h),
    );
    return !moreSpecific;
  };
}
```

Casos cobertos:
- `/configuracoes` (folha) + `/configuracoes/consumo` (folha) → pathname `/configuracoes/consumo`:
  - "Configurações" verifica: existe `/configuracoes/consumo` mais específico que matcha → NÃO ativa.
  - "Consumo IA" verifica: pathname é exatamente `/configuracoes/consumo` → ativa.
- `/usuarios` (folha) + pathname `/usuarios/123/edit`: nenhum href mais específico em NAV_ITEMS → "Usuários" ativa.
- `/relatorios` (com children) + `/relatorios/conversas`: pai ativa porque pathname.startsWith('/relatorios/'); filho "Conversas" ativa via match exato (folha sem mais específicos).

#### 3.11.3 Test connection profundo

`testLlmConnection` retorna estrutura ampliada:

```ts
export interface TestLlmConnectionResult {
  reachable: boolean;
  message?: string;
  /** Se conseguiu validar saldo: true/false. undefined = não verificável. */
  creditOk?: boolean;
  /** USD restantes (best-effort). */
  creditRemainingUsd?: number;
  /** Categorização do erro. */
  errorKind?: "invalid_key" | "model_not_found" | "no_credit" | "rate_limit" | "network" | "other";
  tokensInput?: number;
  tokensOutput?: number;
}
```

Lógica por provider:

- **OpenAI**:
  1. `GET /v1/models` com `Authorization: Bearer KEY`.
  2. Se 401 → `errorKind: "invalid_key"`.
  3. Procurar `model.id === input.model` na lista. Se não → `errorKind: "model_not_found"`.
  4. Validação opcional de saldo: `GET /dashboard/billing/credit_grants` (deprecated mas ainda funciona em algumas contas) ou pular.
  5. Se tudo OK → fazer chamada minimal `chat.completions` com `max_tokens: 16` e validar 200.
  6. Se 429 com erro `insufficient_quota` → `errorKind: "no_credit"`, `creditOk: false`.
- **Anthropic**:
  1. `POST /v1/messages` com `model: input.model`, `max_tokens: 16`, mensagem trivial.
  2. 401 → `invalid_key`.
  3. `model_not_found` (404 ou erro tipo "Invalid model") → `model_not_found`.
  4. 429 + `credit_balance_too_low` → `no_credit`.
  5. 200 → `reachable: true`. Saldo não verificável, `creditOk: undefined`.
- **Gemini**:
  1. `POST https://generativelanguage.googleapis.com/v1beta/models/{model}:generateContent?key=KEY` com input mínimo.
  2. 400 + erro de modelo → `model_not_found`.
  3. 403/401 + `API_KEY_INVALID` → `invalid_key`.
  4. 200 → `reachable: true`. `creditOk: undefined` (Gemini é free tier ou pago).
- **OpenRouter**:
  1. `GET https://openrouter.ai/api/v1/credits` com Bearer KEY.
  2. 401 → `invalid_key`.
  3. Parse `data.balance`. Se 0 → `creditOk: false`.
  4. `POST /api/v1/chat/completions` com `model: input.model`, `max_tokens: 16`.
  5. 404 + `model_not_found` → `model_not_found`.
  6. 200 → `reachable: true`, `creditOk: balance > 0`.

Mensagens (PT-BR) mapeadas a partir de `errorKind`:

- `invalid_key` → "API key inválida ou expirada."
- `model_not_found` → "Modelo \"{model}\" não encontrado neste provedor."
- `no_credit` → "Conexão OK, mas a conta está sem crédito."
- `rate_limit` → "Limite de requisições atingido. Tente novamente em alguns segundos."
- `network` → "Erro de rede ao conectar com o provedor."
- `other` → mensagem original do provider.

#### 3.11.7 Auto-save flow detalhado

```
User clica "Testar conexão":
  → testLlmConnection(input)
  → if reachable && creditOk !== false:
       → saveLlmConfig(input)
       → toast.success("Conexão OK · Configuração salva")
       → router.refresh()
     elif reachable && creditOk === false:
       → toast.warning("Conexão OK, mas sem crédito.")
       → não salva automaticamente, mas mostra botão "Salvar mesmo assim"
     else:
       → toast.error(message mapeado por errorKind)

User clica "Salvar configuração" (manual):
  → testLlmConnection(input)
  → if reachable && creditOk !== false:
       → saveLlmConfig(input)
       → toast.success("Configuração salva")
     elif reachable && creditOk === false:
       → confirm dialog "Conexão OK, mas sem crédito. Salvar mesmo assim?"
       → if confirma → saveLlmConfig(input)
     else:
       → toast.error
       → não salva
```

| Test connection deep pode rodar lento | UI mostra spinner, timeouts de 8 s por provider. |
| Auto-save acidental quando user só queria testar | Apenas auto-saves quando `creditOk !== false` E `reachable === true`. Usuário sempre pode trocar key e re-testar. |

| Catálogo desatualiza | Centralizado em `catalog.ts`; bump trivial. Documentar processo de update no design-system. |
| Drag-and-drop OOS pode frustrar UX | Aceitável: botões up/down + remove cobrem o caso. Doc no tour. |
