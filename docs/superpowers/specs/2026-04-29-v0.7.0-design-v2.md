## 1. Contexto e motivação

(Inalterado da v1)

Após o deploy da v0.6.0 (e patch v0.6.1), o usuário levantou problemas de UX e qualidade no relatório principal (Conversas) e na navegação geral. Esta release é **polimento de qualidade**, sem novos relatórios.

### Princípios

1. Aproveitar telas grandes (até 1600 px de conteúdo).
2. Filtros mais inteligentes, menos grosseiros — mais poder, menos ruído visual.
3. Tour realmente útil e visualmente íntegro.
4. Sincronização real entre toggle Matrix IA e todas as visualizações.
5. Sidebar limpa, sem "borda esquerda violet".
6. Branding mais curto: "Relatórios Inteligentes".

### 3.2 [R2] Sidebar — active state

#### Hoje
```tsx
active
  ? "bg-muted/50 text-violet-500 border-l-2 border-violet-500 pl-[10px]"
  : "text-muted-foreground hover:bg-muted/30 hover:text-foreground"
```
Aplicado idêntico para menu raiz e submenu.

#### Decisão (A1)

**Menu raiz ativo:**
```tsx
active
  ? "bg-violet-500/10 text-violet-600 dark:text-violet-300 hover:bg-violet-500/15"
  : "text-muted-foreground hover:bg-muted/40 hover:text-foreground"
```
Ícone do item ativo: `text-violet-600 dark:text-violet-300`. Sem `border-l-*`. Sem `pl-[10px]`. Padding consistente.

**Submenu ativo:** estrutura visual diferente para criar hierarquia.
- Container: `bg-violet-500/5 text-violet-600 dark:text-violet-300` (fundo ainda mais sutil que o pai).
- Indicador: dot violet 4 px renderizado **antes** do ícone — `<span class="block h-1 w-1 rounded-full bg-violet-500"/>`.
- Hover não-ativo: `text-foreground bg-muted/30`.
- Linha vertical do grupo: `border-l border-border/40` (menos visível).

**Verificação light mode:** `bg-violet-500/10` sobre `bg-background` (branco) deve dar contraste OK; texto `text-violet-600` sobre fundo violet/10 atende WCAG AA. Validar no dev. Se necessário ajustar para `bg-violet-500/15`.

### 3.4 [R4] Pente fino — Conversas

#### 3.4.1 Coluna "Ações" (refator)

- `COLUMNS` ganha `render: (row) => <OpenInChatwoot accountId={accountId} displayId={row.display_id} />` — mas `accountId` não está no escopo de `COLUMNS` (módulo). Solução: passar `accountId` para o array via factory `buildColumns(accountId)`, ou manter array estático e passar `accountId` no `TableCell` via prop especial.
- **Solução**: factory `function buildColumns(accountId: number): ColumnDef[]` chamada uma vez por `useMemo`. Remove o `if (col.key === 'actions')` do body.
- Botão `<OpenInChatwoot>` ganha estilo levemente realçado (`hover:bg-violet-500/10`).

#### 3.4.2 Labels — neutras + todas

- `labels-chips.tsx`:
  - Remover `LABEL_COLORS`, `colorFromName`, `getContrastColor`, `isHex`.
  - Chip neutro: `bg-muted/40 text-muted-foreground border border-border/40 rounded-full px-2 py-0.5 text-[11px]`.
  - Renderizar **todas** as labels.
  - Truncate por chip: `max-w-[140px]`.
  - Container `flex-wrap gap-1`.
  - Tooltip (`title`) do chip = nome completo.
  - Quando 0 labels: `—`.
  - Prop `max?: number` permanece (default `Infinity`) para compatibilidade futura — quando ativo, exibe `+N` neutro como antes.

#### 3.4.3 Atributos — chips chave:valor

- Substituir o pill "N atributos" por lista de chips.
- Função `formatAttribute(key, value): string`:
  - String/number/boolean → `value` (truncado em 80 chars).
  - null/undefined/"" → pulado.
  - Object/Array → `Array.isArray ? '[lista]' : '[obj]'` com tooltip detalhado.
- Render:
  ```tsx
  <div class="inline-flex flex-wrap gap-1">
    {entries.map(([k, v]) => (
      <span class="inline-flex max-w-[180px] items-center gap-1 rounded-md border border-border/40 bg-muted/40 px-1.5 py-0.5 text-[11px]" title={`${k}: ${rawValue}`}>
        <span class="font-medium text-muted-foreground/80 truncate">{k}:</span>
        <span class="text-foreground/80 truncate">{formatAttribute(k, v)}</span>
      </span>
    ))}
  </div>
  ```
- 0 atributos: `—`.
- A coluna `custom_attributes` muda `defaultVisible: false` → mantém-se `false` (usuário ativa via toggle de colunas se quiser ver). Ajuste opcional: subir para `true` se for valioso por padrão; **decisão**: manter `false` para não poluir mais; o usuário pode ativar.

#### 3.4.4 Ordenação — descoberta

- Header continua `<button>` clicável; tooltip já existe.
- Adicionar **chip "Ordenação · N"** no toolbar de filtros (ver §3.6).
- Quando há sort ativo, badge no chip; clique abre Drawer de Ordenação.
- Comando **shift+click** documentado no tour (etapa 4).
- Manter botão "Limpar ordenação" no toolbar interno da tabela (já existe), redundância intencional.

#### 3.4.5 Empty state melhorado

- Atual: ícone Inbox + "Nenhuma conversa encontrada" + dica.
- Novo: se há filtros aplicados (`filterState !== EMPTY_FILTER_STATE`), mostrar botão `[Limpar filtros]` no empty state que dispara reset.
- Se não há filtros (busca ampla retorna 0): manter mensagem genérica.

#### 3.4.6 Investigação "0 conversas / dessincronizado"

- Subagent dedicado durante implementação:
  1. Logar como super_admin, sem filtros, "Todos" o período, fetch.
  2. Comparar com `SELECT count(*) FROM conversations WHERE account_id = $accountId AND inbox_id <> 31;` direto no banco (read-only).
  3. Se contagens divergem: bug. Investigar `fetchConversas` action, `resolvePeriod`, cache.
  4. Se contagens batem: não é bug, é apenas filtro restritivo. Documentar.
- Fix se for bug: prioridade alta dentro do plan.

### 3.5 [R5] MultiSelect — busca interna + Selecionar todos

#### Componente

- Refatorar `MultiSelectCheckbox` em `advanced-filters.tsx` (ou extrair para `src/components/ui/multi-select-checkbox.tsx`).
- Estado interno: `searchQuery` string.
- Lista filtrada: `filteredOptions = options.filter(o => o.name.toLowerCase().includes(searchQuery.toLowerCase()))`.
- Header do popover/seção:
  - Input busca: `<Input placeholder="Buscar..." value={searchQuery} onChange={...} className="h-8" />` com ícone Search à esquerda.
  - Linha de ações: `[Selecionar todos] [Limpar tudo]`. Comportamento:
    - Sem busca + nenhum selecionado: "Selecionar todos" → `onChange(options.map(o => o.id))`.
    - Sem busca + alguns/todos selecionados: "Limpar tudo" → `onChange([])`.
    - Com busca + nenhum dos visíveis selecionado: "Selecionar visíveis" → `onChange(uniq([...value, ...filteredOptions.map(o => o.id)]))`.
    - Com busca + alguns dos visíveis selecionados: "Limpar visíveis" → `onChange(value.filter(id => !filteredOptions.find(o => o.id === id)))`.
  - Tooltip nos botões: "Aplica somente aos resultados visíveis" (quando busca ativa).
- Lista vazia (busca sem resultados): `<p>Nenhum resultado para "{query}"</p>`.

#### Relabel

- Em `advanced-filters.tsx`, label `"Equipe"` → `"Departamento"`.
- Em `conversas-tour.ts` etapa de filtros: copy `"caixas, equipes, atendentes"` → `"caixas, departamentos, atendentes"`.

#### 3.6.1 Toolbar (linha sempre visível)


- Linha 3 (condicional): chips de filtros aplicados + "Limpar tudo".

#### 3.6.2 Chip "Filtros · N"

- Componente botão/chip:
  - Label: `Filtros` + badge `· N` quando `appliedCount > 0`.
  - Estado pendente (drawer alterou mas não aplicou): dot violet pequeno no canto superior direito do chip.
  - Aria-label dinâmico: `"Abrir painel de filtros · {N} aplicados, {M} pendentes"`.
- Clique: abre `<FiltersDrawer>`.

#### 3.6.3 `<FiltersDrawer>`

- Sheet lateral direita (componente `Sheet` do base-ui ou shadcn-pattern).
- Largura: 360 px desktop; **fullscreen em viewport < 640 px**.
- Composição:
  - **Header**: `Filtros` + close button + "{N} pendente(s)" se há diff.
  - **Body**: lista de seções `<Collapsible>`. Cada uma expande para mostrar `MultiSelectCheckbox` em modo **inline** (não popover). Default: colapsadas; expandem ao clicar; estado lembrado durante a sessão (`useState`).
  - Seções (na ordem): Caixa de entrada, Departamento, Atendente, Status, Prioridade.
  - Cada header da seção: `[ícone] Nome [chevron] [contador]`.
  - **Footer (sticky)**: `[Limpar todos] [Aplicar (N)]`. "Aplicar" desabilitado se `!isDirty`.
- Comportamento de aplicação:
  - Drawer aberto: alterações ficam em `draft` interno do drawer, não tocam URL.
  - "Aplicar": promove `draft` → `applied` → push URL → fecha drawer.
  - ESC ou close: descarta `draft` (volta para `applied`).
  - Clicar fora (overlay): igual ESC.
- Focus trap: ao abrir, foco vai para "Limpar" / primeiro elemento; Tab cicla dentro do drawer; ESC fecha.

#### 3.6.4 Chip "Ordenação · N"

- Mesmo padrão do chip Filtros, mas para `sortStack`.
- N = quantidade de critérios.
- Clique: abre `<SortingDrawer>`.

#### 3.6.5 `<SortingDrawer>`

- Sheet lateral direita, mesmas dimensões.
- Header: `Ordenação` + close + contador.
- Body: lista de critérios atuais.
  - Cada item: `[≡ drag] [Coluna ▾ select] [Asc/Desc toggle group] [×]`.
  - Drag-and-drop por mouse (HTML5 DnD ou framer-motion `Reorder.Group`).
  - Select de coluna: opções = colunas com `sortable: true` que ainda não estão na pilha.
  - Toggle Asc/Desc: dois botões com ícones `ChevronUp` / `ChevronDown` selecionados.
- Botão `[+ Adicionar critério]`: cria nova entrada no fim da pilha.
- Footer: `[Limpar ordenação] [Aplicar]`.
- Apply: promove draft → real `sortStack` → não recarrega da rede (sort é client-side em `ConversasTable`).

#### 3.6.6 `<AppliedFiltersChips>`

- Renderizado abaixo da toolbar quando há `applied !== EMPTY`.
- Lista chips por categoria com seleção:
  - `Departamento: 3` (ou `Departamento: Suporte`, se 1 só).
  - X remove **toda** a seleção daquele grupo.
- Botão "Limpar tudo" no fim: dispara reset.
- Mostrar até 6 chips inline; se > 6: `[+N mais]` que abre o drawer.

### 3.8 [R8] Sincronização Matrix IA — auditoria

#### 3.8.1 Helper canônico

```ts
// src/lib/chatwoot/queries/meta-cache-for-user.ts
import type { AuthUser } from "@/lib/auth-helpers";
import { getInboxes, type MetaItem } from "./meta-cache";
import { getMatrixIAIncluded } from "@/lib/reports/matrix-ia-setting";

const MATRIX_IA_INBOX_ID = 31;

export async function getInboxesForUser(
  accountId: number,
  user: Pick<AuthUser, "platformRole">,
): Promise<{ data: MetaItem[]; stale?: boolean }> {
  const result = await getInboxes(accountId);
  if (user.platformRole === "super_admin") return result;
  const includeMatrixIA = await getMatrixIAIncluded();
  if (includeMatrixIA) return result;
  return {
    ...result,
    data: result.data.filter((inbox) => inbox.id !== MATRIX_IA_INBOX_ID),
  };
}
```

## 5. Riscos e mitigações

| Risco | Mitigação |
|-------|-----------|
| Drawer mobile + bubble Nex z-index conflito | Drawer z=1900, overlay 1890; bubble fica em z=2000 e tem `pointer-events: auto` apenas em hover. Validar empilhamento. |
| `getInboxesForUser` quebra cache compartilhado | Cache continua de `getInboxes` (sem filtro); filtro acontece em memória após retorno. Sem invalidação extra. |
| `max-w-[1600px]` "afina" em telas 1920×1080 | 1600 px conteúdo + 160 padding cada lado em 1920 = OK. Em 1366×768 (notebook), padding cai para 16 px (já é mobile-style). |
| Tour overlay ainda quebrar em viewport 1280×720 | Testar matriz: 1280, 1440, 1920, 2560, mobile 390. |
| Inbox 31 escondida confunde admin | Toast já avisa após toggle. Adicionar nota no toggle: "Afeta dropdowns, tabelas e gráficos." |
| Refator de `AdvancedFilters` quebra `mensagens-nao-respondidas` | Manter API pública estável; testes de integração em ambas as páginas. |
| Light mode contraste violet/10 | Validar com DevTools color picker; ajustar para violet/15 se necessário. |
| Página `/configuracoes` quebra (ela é `narrow`?) | Usar `PageShell variant="narrow"` em todas as páginas de admin. Testar manualmente. |
| Drag-and-drop em `<SortingDrawer>` | Usar framer-motion `Reorder.Group` (já estamos com framer); fallback teclado: `Move up/down` buttons em cada item. |
